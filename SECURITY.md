# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability affecting the project's build
scripts, automation, dependencies or repository infrastructure, please
report it privately instead of opening a public issue.

When reporting a vulnerability, please include:

- a description of the issue;
- its potential impact;
- steps to reproduce the problem;
- any suggested mitigation or fix, if known.

Thank you for helping improve the security of AI Futures Guide.
